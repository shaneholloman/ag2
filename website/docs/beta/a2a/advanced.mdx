Topics that aren't part of the day-to-day A2A path: human-in-the-loop, transparent reconnects on streaming drops, plugging in a custom executor, and the error hierarchy.

## Human-in-the-Loop via `input_required`

When a server-side executor raises `#!python requires_input(...)` mid-task, the A2A server transitions the task to `#!python TASK_STATE_INPUT_REQUIRED` and surfaces the prompt to the client. The AG2 client invokes the local agent's `#!python hitl_hook` with the prompt and continues the same turn with the reply — the server prompt does **not** leak into the final response text.

```python linenums="1" hl_lines="4-5 10"
from autogen.beta import Agent
from autogen.beta.a2a import A2AConfig

async def hitl_hook() -> str:
    return input("server asks input> ")

remote = Agent(
    "remote",
    config=A2AConfig(card_url="http://127.0.0.1:8000"),
    hitl_hook=hitl_hook,
)
reply = await remote.ask("start")  # server may request input multiple times
print(reply.response.content)
```

`#!python A2AConfig.input_required_timeout` caps how long the client waits on the hook. `#!python None` (default) waits indefinitely — which matches the behaviour of `#!python ConversationContext.input` in regular agents.

See [Human in the Loop](/docs/beta/context/human_in_the_loop) for the underlying `#!python hitl_hook` contract.

## Streaming Reconnect

Streaming connections drop. Network blips, load balancer recycles, idle timeouts — the SSE channel breaks, and the client needs to recover without losing partially-streamed artifacts.

The A2A client keeps internal drive state across drops: on `#!python A2AClientError` mid-stream it issues a fresh `#!python subscribe` against the same `#!python task_id`, deduplicates artifacts already seen by `#!python artifact_id` (and messages by `#!python message_id`), and continues from where it failed. Application code sees a single uninterrupted reply.

```python linenums="1" hl_lines="6-7"
remote = Agent(
    "remote",
    config=A2AConfig(
        card_url="http://127.0.0.1:8000",
        streaming=True,
        max_reconnects=3,
        reconnect_backoff=0.5,
    ),
)
reply = await remote.ask("a long answer")
```

| Field | Default | Purpose |
|---|---|---|
| `#!python max_reconnects` | `#!python 3` | Total reconnect attempts before giving up with `#!python A2AReconnectError` |
| `#!python reconnect_backoff` | `#!python 0.5` | Seconds to wait between attempts (constant — no jitter or exponential) |

When attempts are exhausted the client raises `#!python A2AReconnectError(attempts=N)`. Catch it to fall back to a polling re-fetch via `#!python get_task` if you need recovery beyond the streaming budget.

## Custom `AgentExecutor`

`#!python A2AServer` defaults to wrapping the supplied `#!python Agent` in AG2's standard `#!python AgentExecutor`. When you need behaviour that doesn't fit `#!python Agent.ask` — a HITL-first turn, a multi-agent pipeline, a non-standard task lifecycle — drop in your own executor:

```python linenums="1" hl_lines="1 5-7 9-10 12"
from a2a.server.agent_execution import AgentExecutor as A2AAgentExecutorBase

from autogen.beta.a2a import A2AServer

class MyExecutor(A2AAgentExecutorBase):
    async def execute(self, request_context, event_queue) -> None:
        ...

    async def cancel(self, request_context, event_queue) -> None:
        ...

server = A2AServer(agent_stub, executor=MyExecutor())
```

The executor owns the entire request lifecycle: parsing the inbound message, emitting status updates and artifacts to the `#!python event_queue`, and signalling terminal state. The wrapped agent passed to `#!python A2AServer(agent_stub, ...)` is still used for card metadata (`#!python name`, `#!python description`, etc.) — make it a stub if a real agent doesn't fit your design.

## Errors

All A2A errors live in `#!python autogen.beta.a2a.errors` and inherit from `#!python A2AError`. Catch `#!python A2AError` for everything; catch the specifics when you want to react differently.

| Exception | Raised when |
|---|---|
| `#!python A2AError` | Base class — catch this for any A2A failure |
| `#!python A2AInvalidCardError` | The card is missing data required to connect (no `#!python supported_interfaces`, no usable URL) |
| `#!python A2AClientToolsNotSupportedError` | Client passed `#!python tools=` but the server card doesn't advertise the `#!python urn:ag2:client-tools:v1` extension |
| `#!python A2AReconnectError` | Streaming reconnect attempts exhausted — `#!python err.attempts` holds the count |
| `#!python A2ATaskTerminalError` | Base for the three terminal-state errors below; carries `#!python err.task` with the final `#!python Task` (status, history, artifacts) |
| `#!python A2ATaskFailedError` | Task ended in `#!python TASK_STATE_FAILED` |
| `#!python A2ATaskRejectedError` | Task ended in `#!python TASK_STATE_REJECTED` |
| `#!python A2ATaskAuthRequiredError` | Task ended in `#!python TASK_STATE_AUTH_REQUIRED`. Per A2A spec §7.6 the agent expects credentials out-of-band — apply them and retry |

Catch `#!python A2ATaskTerminalError` to handle any terminal failure uniformly; switch on the concrete subclass when the recovery path differs (e.g. retry with auth on `#!python AuthRequired`, surface the error to the operator on `#!python Failed` / `#!python Rejected`).

## Testing Helpers

`#!python autogen.beta.a2a.testing` provides utilities for in-process A2A tests — no real socket, no port binding.

```python linenums="1" hl_lines="6-7"
from autogen.beta import Agent
from autogen.beta.a2a import A2AConfig, A2AServer
from autogen.beta.a2a.testing import make_test_client_factory

server = A2AServer(agent)
factory = make_test_client_factory(server, url="http://test")
remote = Agent("remote", config=A2AConfig(card_url="http://test", httpx_client_factory=factory))
await remote.ask("ping")
```

| Helper | Purpose |
|---|---|
| `#!python make_test_client_factory(server, url=..., timeout=...)` | `#!python httpx.AsyncClient` factory dispatching JSON-RPC into the server's ASGI app via `#!python httpx.ASGITransport` |
| `#!python make_test_rest_client_factory(server, url=..., timeout=...)` | Same idea for REST — builds the REST app and a card declaring only the REST interface |
| `#!python pick_free_port(host="127.0.0.1")` | Probe a free TCP port. Used by gRPC tests since gRPC has no in-process transport equivalent |

```python linenums="1"
from autogen.beta.a2a.testing import pick_free_port

port = pick_free_port()
grpc_server = server.build_grpc(bind=f"127.0.0.1:{port}", grpc_url=f"grpc://127.0.0.1:{port}")
```
